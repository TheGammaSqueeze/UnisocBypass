# Uboot permanent-unlock patch

`tools/patch_uboot_unlock.py` applies seven binary patches to a stock UMS512 uboot image so the bootloader permanently reports itself as unlocked with AVB disabled, and the boot-time "SKIP VERIFY" warning and its delay are removed.

## Prerequisite

The device must already have a patched SPL from `tools/patch_spl.py` + `scripts/flash_spl.sh`. Without that, SPL will RSA-verify the modified uboot and refuse to boot (hang in infinite loop).

## What it does

Seven targeted instruction replacements:

| Code offset | Original | New | Effect |
|-------------|----------|-----|--------|
| `0x78c98` | `cbz w0, +12` (`0x34000060`) | `b +12` (`0x14000003`) | Lock-state parser always falls through to "unlocked" branch, writing 1 to the state variable at `0x22d7e0` |
| `0x20330` | `adrp x0, 0x230000` (`0x90001080`) | `mov w0, #0` (`0x52800000`) | Overwrites first instruction of the SKIP_VERIFY print + 6-second-delay function |
| `0x20334` | `stp x29,x30,[sp,#-32]!` (`0xa9be7bfd`) | `ret` (`0xd65f03c0`) | Combined with above: function returns 0 immediately, no text printed, no delay |
| `0x20288` | `adrp x0, 0x96000` (`0xd00003a0`) | `nop` (`0xd503201f`) | INFO LOCK FLAG IS UNLOCK print call (1 of 3) |
| `0x2028c` | `add x0, x0, #0x639` (`0x9118e400`) | `nop` (`0xd503201f`) | INFO LOCK FLAG IS UNLOCK print call (2 of 3) |
| `0x20290` | `bl printf` (`0x97ffbba4`) | `nop` (`0xd503201f`) | INFO LOCK FLAG IS UNLOCK print call (3 of 3) |
| `0x20fec` | `ldr w1, [x0]` (`0xb9400001`) | `mov w1, #2` (`0x52800041`) | verifiedbootstate always evaluates to the state-2 ("orange") branch regardless of what is stored at `0x23080c` |

After patching, the tool updates the DHTB SHA256 and SIMGHDR data hash via `rehash.py`.

## Observable effects

In `/proc/cmdline` on Android:

```
androidboot.verifiedbootstate=orange
androidboot.flash.locked=0
androidboot.vbmeta.device_state=unlocked
```

From `fastboot getvar`:

```
unlocked: yes
is-userspace: yes
```

Boot behavior:
- No SKIP VERIFY warning text shown on display
- No 6-second warning delay
- No INFO LOCK FLAG IS UNLOCK text shown
- AVB is skipped (uboot believes bootloader is unlocked, per its own stock "if unlocked, skip verify" branch)

## Usage

```bash
# Produce the patched image
python3 tools/patch_uboot_unlock.py backups/uboot_b.bin my_unlocked_uboot.img

# Verify the hash is valid (optional sanity check)
python3 tools/verify_image.py my_unlocked_uboot.img

# Flash to both slots (requires patched SPL already in place)
./scripts/flash_uboot.sh my_unlocked_uboot.img
adb reboot
```

## Dry-run

To confirm all 7 patch points match the expected original instructions without writing:

```bash
python3 tools/patch_uboot_unlock.py backups/uboot_b.bin /dev/null --dry-run
```

This is useful when applying to an uboot from a firmware version other than the one this was developed against. If any pattern mismatches, the tool aborts with a list of which patches did not apply, so you can investigate or disable that patch.

## Combining with other modifications

`patch_uboot_unlock.py` only changes the 7 listed offsets. You can apply other modifications before or after with `modify_uboot.py`, `rehash.py`, or your own scripts. The final file's DHTB hash will need to be regenerated by whichever tool touches the code last (all of them call `rehash()` automatically).

## Reverting

Flash the stock uboot backup:

```bash
./scripts/flash_uboot.sh backups/uboot_b.bin
adb reboot
```

(This assumes `scripts/backup.sh` was run before making changes, which creates `backups/uboot_b.bin` from the device.)

## Why AVB is actually skipped

Stock uboot already contains a branch that skips AVB when the bootloader is unlocked. That branch is gated by a call to the lock-state read (our patch 1 forces its output to 1). The stock code path then prints a warning and the SKIP VERIFY text, which our patches 2-6 silence. The AVB skip itself is not something we disable explicitly; we just make the normal unlocked-path behavior happen unconditionally and invisibly.

In short: uboot already knows how to skip AVB when unlocked. We just make it believe it is always unlocked, quietly.
